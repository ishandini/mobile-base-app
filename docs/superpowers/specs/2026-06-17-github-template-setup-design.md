# GitHub Template Setup Design

**Date:** 2026-06-17
**Status:** Approved

## Goal

Convert the current Flutter base app (on `feature/template`) into a clean GitHub template repository with no project-specific branding, credentials, or git history.

## Scope

Three phases: code cleanup → history wipe → GitHub template setup.

---

## Phase 1 — Code Cleanup (on `feature/template`)

### Android — `android/app/build.gradle`

| Flavor | Current app_name | Target app_name | CleverTap |
|--------|-----------------|-----------------|-----------|
| `dev`  | `"Prime Dev"`   | `"Base App Dev"` | Remove `manifestPlaceholders` block |
| `qa`   | `"Prime QA"`    | `"Base App QA"`  | Remove `manifestPlaceholders` block |
| `prod` | `"Prime Edge"`  | `"Base App"`     | Remove `manifestPlaceholders` block |

Each `productFlavors` entry currently has:
```gradle
manifestPlaceholders = [
    CLEVERTAP_ACCOUNT_ID: "...",
    CLEVERTAP_TOKEN: "..."
]
```
Remove these blocks entirely from all three flavors.

### iOS — `ios/Runner/Info.plist`

- Change `CFBundleName` value from `"Prime Edge"` → `"Base App"`
- Remove the `CleverTapAccountID` key-value pair
- Remove the `CleverTapToken` key-value pair

### iOS — `ios/Runner.xcodeproj/project.pbxproj`

Change `PRODUCT_NAME` in all build configurations:
- `"Prime Edge"` → `"Base App"` (PROD Debug + Release)
- `"Prime QA"` → `"Base App QA"` (QA Debug + Release)
- `"Prime Dev"` → `"Base App Dev"` (DEV Debug + Release)

Remove `CLEVERTAP_ACCOUNT_ID` and `CLEVERTAP_TOKEN` build setting lines from all four configuration blocks (8 lines total).

### .env files (local only — gitignored, not committed)

Reset `env/.env.dev`, `env/.env.qa`, `env/.env.prod` to placeholder values:
- Remove `CLEVERTAP_ACCOUNT_ID` and `CLEVERTAP_TOKEN` lines
- Replace all real URLs and credentials with `''` or `your_value_here`
- Structure should mirror `env/.env.sample`

`env/.env.sample` already has no CleverTap keys — no change needed there.

---

## Phase 2 — Git History Wipe

After all code changes are committed to `feature/template`:

1. `git checkout main`
2. `git checkout --orphan fresh-main` — new branch with no parent commits
3. `git add -A` — stage all current files (reflecting `feature/template` state merged in)
4. Single initial commit: `chore: initial Flutter base app template`
5. Delete old `main`: `git branch -D main`
6. Rename: `git branch -m main`
7. `git remote remove origin` — sever link from `prime-land-mobile`

Result: `main` has exactly one commit, no history, no remote.

---

## Phase 3 — GitHub Template Setup

Steps performed by the developer (not automated):

1. Create a new GitHub repository (e.g., `flutter-base-app` or `mobile-base-app`)
2. `git remote add origin <new-repo-url>`
3. `git push -u origin main`
4. In GitHub repo **Settings → General** → enable **"Template repository"**

---

## Out of Scope

- Removing or renaming other Prime Group branding in source code (company URLs in usage description strings, Facebook App ID, etc.) — these are acceptable as placeholder examples for template users to replace
- Removing the `ios/Pods/` CleverTap SDK references — these are regenerated by `pod install` and not committed
- Deleting local feature branches — optional, not required for the template

---

## Success Criteria

- `flutter analyze` reports 0 errors after changes
- Android flavor app names display "Base App", "Base App Dev", "Base App QA"
- iOS scheme names display "Base App", "Base App QA", "Base App Dev"
- No `CLEVERTAP_ACCOUNT_ID` or `CLEVERTAP_TOKEN` in any tracked file
- `git log` on `main` shows exactly 1 commit
- `git remote -v` returns empty
- `env/.env.dev`, `.env.qa`, `.env.prod` contain only placeholder values
